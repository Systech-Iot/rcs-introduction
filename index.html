PRODUCT VALUE

把现场搬运任务转化为可调度、可监控、可追溯的运行体系

RCS 将车辆、地图、订单、路径和交通资源统一纳管，既满足调度员日常操作，也支撑 WMS / MES / WCS 等外部系统集成。

01

标准接入

以 VDA5050 + MQTT 为通信基础，统一接收车辆状态、位置、电量、连接与动作执行信息。

02

智能调度

通过路径规划、点位锁定、安全半径、交通避让和任务续接，支撑多车稳定协同运行。

03

实时可视

地图看板实时展示 AGV、订单、锁点、避让关系与运行状态，异常现场更容易定位。

04

交付验证

内置虚拟 AGV、地图模拟和随机 WMS 订单，支持项目前期演示、调试和交付前验证。

SYSTEM ARCHITECTURE

前端、网关、调度核心、通信协议与仿真服务分层解耦

系统采用前后端分离和微服务转发架构，对外提供 HTTP API、SSE 实时推送与 Swagger 文档，对内通过 TCP 服务承载核心调度能力。

调度核心

AGV / AMR

CORE CAPABILITIES

覆盖 RCS 从车辆接入到现场运营的核心功能

士腾 RCS 已形成车辆、车型、Action、地图、路径、订单、看板、货架、充电停车和仿真的完整主链路。

核心能力

AGV 接入与车辆管理

统一管理车辆台账、车型参数、地图绑定、启停状态和关键控制操作。

  • VDA5050 v2、MQTT、连接状态与心跳检测
  • 车辆创建、批量创建、启停、删除保护和作废恢复
  • 电量阈值、速度、载重、尺寸、安全半径和坐标误差配置
核心能力

地图建模与可视化编辑

支持惯导和 SLAM 地图，完成点位、路段、业务点位和运行图模型维护。

  • 地图创建、编辑、启停、预览、文件上传和静态访问
  • 行驶点、充电点、工位、停车点、交通管制点配置
  • 点位/路段批量操作、曲线路段、交叉点、速度和权重管理
核心能力

路径规划与交通控制

通过图搜索、锁点和避让机制，让多车在同一地图中稳定协同。

  • A* 路径规划、动态路段权重、路径续接和等待处理
  • 点位锁定、安全半径锁定、交叉点锁定和点位释放
  • 避让关系、避让订单和阻塞缓解机制
核心能力

订单调度与 WMS 集成

承接人工任务和外部系统任务，并转化为 AGV 可执行的标准订单。

  • 用户订单、WMS 订单、系统订单和订单复制
  • 指定 AGV 或按车型自动分配，支持优先级调度
  • 前置订单、避让订单、充电订单、停车订单和状态流转
核心能力

Action 动作编排

把取放、充电、暂停、定位、等待触发等现场动作编排进任务流程。

  • 标准 VDA5050 Action 与自定义 Action 并存
  • 动作参数规则、阻塞类型、适用范围和执行历史
  • 支持即时动作、点位动作、路段动作、起终点动作
核心能力

自动充电与自动停车

基于电量、车辆状态和点位资源自动生成充电或停车任务。

  • 低电量判断、自动充电订单和指定充电点
  • 充电动作编排和充电中接单控制
  • 空闲停车、指定停车点和停车点优先级
核心能力

实时看板与历史回溯

调度员可以实时查看车辆、订单、地图和交通状态，并进行异常复盘。

  • 地图看板、AGV 列表、订单统计和状态图表
  • SSE 推送 state、visualization、connection 和实时看板数据
  • 锁点、避让信息、历史轨迹和路径回溯
交付支撑

仿真验证与演示

在没有真实车辆或项目早期阶段，也能验证地图、调度和订单链路。

  • 虚拟 AGV 启停、批量模拟和状态发布
  • 随机 WMS 订单、随机订单配置和地图模拟
  • 基于 MQTT 的 VDA5050 仿真闭环
基础能力

开放接口与基础配置

为前端管理端、外部系统、现场对象和交付工具提供统一支撑。

  • HTTP API、TCP 微服务转发、Swagger 文档和 CORS
  • 地图文件、货架台账、工位和动作参数基础配置
  • 用户登录、权限枚举、前端菜单权限和日志查询基础

RUNNING LOOP

从任务下发到车辆执行，形成可解释的调度闭环

RCS 将外部任务转为路径、点位、路段和现场动作，运行中持续维护车辆状态、锁点、避让关系和订单进度，让现场调度可视、可控、可复盘。

实时交通状态

蓝色为当前订单路径，橙色为避让/等待路径，黄色点位表示当前锁定的交通资源。

1

地图与车辆准备

维护 SLAM / 惯导地图、点位/路段拓扑、车型参数、车辆台账和运行阈值。

2

订单生成与分配

支持人工创建、WMS 接入、路线编辑和系统自动任务，按车辆状态和任务优先级分配。

3

路径规划与交通控制

基于地图图模型规划路径，结合锁点、安全半径和避让订单降低冲突风险。

4

执行监控与异常处理

实时接收车辆状态、动作执行、连接状态和可视化位置，订单全过程可追踪。

5

历史回溯与持续优化

通过订单详情、轨迹回放、日志与统计数据复盘运行瓶颈，持续优化现场效率。

SIMULATION DEMO

用仿真视频展示多车调度与避让逻辑

视频展示了系统在多车运行、路径冲突、点位占用和动态避让场景中的调度表现，可用于方案交流、方案评审和项目交付前验证。

百车调度仿真

验证大规模订单与车辆协同压力下的调度链路

通过百车仿真观察车辆分布、路径运行、任务压力和调度节奏，辅助评估地图容量、交通瓶颈和项目规模化部署策略。

避让逻辑演示

展示路径冲突识别、让行调度与通行恢复

在车辆运行过程中识别潜在冲突，通过锁点、安全半径和避让订单协调车辆通行，降低拥堵和死锁风险。
