01RCS 能做什么

RCS 可以理解成现场的「机器人调度台」。您在电脑上操作这个系统，就能指挥厂区里的 AGV（自动搬运车）： 它们走哪条路、去哪个点、去充电还是停车、当前任务做到哪一步。

您日常主要做这些事
  1. 01准备地图
  2. 02登记并启用车辆
  3. 03下发任务
  4. 04看板监控
  5. 05查订单 / 日志

运输、充电、停车都从订单下发；出问题再看订单详情或日志。

请先记住这一条
先准备好（缺一不可）
已启用的地图 + 已启用的车辆
↓ 后面下单和监控才会顺利

必须先有一张已启用的地图，再有一台已启用的车辆，后面下单和监控才会顺利。顺序颠倒时，系统往往会拦截该操作。

02界面说明

登录成功后，页面上方是系统名称，左边是菜单，右边是当前功能页面。 日常操作请从左侧菜单进入对应模块。

左边菜单功能说明
首页欢迎页，没有业务操作，可忽略
AGV管理管理车辆、车型、动作模板；进入车辆详情进行控制
地图管理新建/编辑地图，启用或停用地图
订单管理查询任务、新建任务、取消或复制任务
货架管理登记货架型号资料（尺寸、载重等）
地图看板全屏大屏，实时查看车辆在地图上的位置与状态
日志管理按时间查看系统记录，方便事后排查
小提示： 打开「地图看板」、编辑/预览地图、进入用户管理时，浏览器往往会再弹出一个新窗口。 若点击后无响应，请查看浏览器地址栏附近是否提示「已拦截弹窗」，允许后重试。

账号相关功能不在左边菜单里，而在页面右上角头像中（只有管理员一类账号才会看到）。

03登录与退出

登录方法
  1. 用浏览器打开现场提供的系统地址。
  2. 在「欢迎登录」页输入账号：一般是 11 位手机号。
  3. 输入密码：需要同时有数字和字母，长度 8～32 位。
  4. 点击「登录」进入系统。

演示或交付环境里，常用账号示例如下（具体以现场开通为准）：

账号角色适用对象
17795790955管理员全部功能，含账号权限管理
17795790956地图操作员主要做地图相关工作
17795790957订单操作员主要做订单相关工作

登录页可能已预填管理员账号。首次打开时也可能自动进入系统，属于正常现象。

退出方法
  1. 点击页面右上角头像（或「退出登录」按钮）。
  2. 选择「退出登录」。
  3. 返回登录页后，需再次点击「登录」方可进入（不会立即自动重新登录）。
若点击某菜单提示无权限，说明当前账号不具备该功能权限，请联系管理员开通。

04首次使用流程

新现场、新仓库，或首次建图时，建议严格按以下顺序操作，以减少返工：

① 新建并画好地图 → ② 启用地图 → ③ 配好车型 → ④ 把 AGV 登记进系统 → ⑤ 车联网后启用车辆 → ⑥ 下发第一笔订单 → ⑦ 打开看板持续监控
对照检查：
· 地图仍为「未启用」→ 无法进入看板，任务也难以正常调度。
· 车辆尚未「启用」→ 即使已创建订单，也常常无法正常执行。

05地图：建好并启用

入口：左边菜单 地图管理 → 地图列表

地图是 AGV 可通行的「厂区道路图」。若无地图，车辆将无法定位与路径规划。

5.1 新建一张地图

操作步骤
  1. 点击「新增地图」。
  2. 填写地图名称、描述，并选择类型：
    • 惯导地图：场地比较规则、按格子规划时常用；需要填地图长、宽。
    • SLAM地图：根据现场扫图结果使用；需要上传数据包（.pgm）和 YAML 文件，并填写最小边距。
  3. 填写「边默认最大速度」（路径上的默认最高速度）。
  4. 按需设置墙体安全距离（车辆与墙体之间的安全余量）。
  5. 确认提交。成功后会提示「添加地图成功！」

新建地图默认为未启用。请继续进入编辑，完善点位与路径后再启用。

5.2 编辑地图（绘制点位与路径）

操作步骤
  1. 在列表里找到目标地图（必须是「未启用」才能编辑）。
  2. 点击「编辑」，系统通常会打开新窗口。
  3. 在编辑器里添加/调整点位和路径，并保存。

点位类型选择（关键）

点类型适用场景
行驶点普通途经点，路径中最常见
充电点车辆充电位置；未配置时，「去充电」将无法选择目标点
停车点车辆空闲停靠位置；未配置时，「去停车」将无法选择目标点
工位装卸货、作业的位置

充电点、停车点、工位通常还要填好角度、实物位置等信息，否则保存或启用时可能校验不通过。

编辑时常用能力

  • 框选多个点/边：框选后可批量设置点属性、边属性，或批量启用/禁用点。
  • 点属性：可设置角度、停车优先级、点上要执行的动作；还可「指定车型」—— 不指定表示所有车都能用这个点；指定后，只有对应车型能用，其他车型下单时会看到「车型禁用点」。
  • 边（路线）属性：可设置是否启用、权重（影响系统更倾向选哪条路）、最大速度、边上动作。 图上线的粗细/颜色往往对应速度档（默认/慢速/快速）。
  • 惯导地图：可设置基准点、开关网格/坐标轴/图示；支持撤销（Ctrl+Z）。
  • SLAM地图：一般要先上传/更新扫图文件才能画；可用「选择标定 AGV」； 右键可删除或禁用点/边；边还可以做「平滑曲线」。
编辑完成后请务必点击「保存」。若提示「编辑未保存」或「存在未通过校验的点位」，请处理完毕后再关闭窗口，否则改动可能丢失。

5.3 预览、启用、停用、删除

  • 预览：只看不改，适合核对布局是否正确。
  • 启用：列表里把开关拨到「启用」。启用后，这张图才能用于看板和下发任务。
  • 停用：要改图时，先停用再编辑。如果还有启用中的车，或还有没做完的订单，系统会不允许停用。
  • 删除：只能删除未启用的地图，并且不能还有关联车辆或订单。
操作顺序：修改地图 → 先停用 → 再编辑并保存 → 再启用。已启用的地图，列表中通常不显示编辑、删除按钮。

06车辆：登记并启用

入口：左边菜单 AGV管理

本模块主要包含三项：车型模板、动作模板、以及各台车辆。

6.1 先配车型（建议先做）

进入 车型列表，点击「新增车型」。 车型用于定义该类车辆的外形尺寸、速度与电量策略等参数。 后续登记车辆、创建订单均需选择车型，建议先完成配置。

常见填写项：车型名称、长宽高、最大负载、速度、充电相关电量、允许偏差、控制器型号等。

6.2 再配动作模板（按需）

进入 Action列表。 这里维护车辆能做的动作（例如顶升、某个业务动作）。 下单时可在起点/终点配置这些动作；车辆详情中也可立即下发单个动作。

6.3 登记一台 AGV

操作步骤
  1. 进入 AGV列表，点击「新增AGV」。
  2. 填写序列号、车型、IP、端口、所属地图、导航类型（惯导或 SLAM）等。
  3. 按需开启「自动工作许可」，并确认电量相关参数。
  4. 提交。成功会提示「添加AGV成功！」

列表里还可以：按地图/序列号查询；批量导入、导出；勾选多台车后批量启用/停用、允许自动/禁止自动。 在线状态一般会显示：在线、离线、异常断线。

电量参数说明（编辑车辆时常见）

界面名称说明
自动充电电量电量低于该值时，系统倾向于安排车辆充电（常见默认约 20%）
充电时最低工作电量充电过程中，电量至少达到该值后更适合再接新任务（常见默认约 80%）
充电完成电量空闲时判断是否继续充电的参考阈值（常见默认约 100%）

角度允许偏差、XY 允许偏差：车辆到位时允许的偏差范围；设置过严可能导致频繁判定未停准。

6.4 启用车辆（关键一步）

启用前请现场确认以下三项均已满足：

  1. 车已经绑定到某张地图，并且这张地图已经启用；
  2. 车已经联网上线（列表里能看到在线）；
  3. 车已经上报位置，并且停在地图的某个点上。

任一条件不满足时，启用往往会失败。

「自动工作许可」开关请认真对待：
· 开启：空闲时系统可自动向车辆派发任务，也可能自动安排充电/停车。
· 关闭：通常不会自动安排充电/停车；通过「新增 WMS 订单」下发的任务，也可能无法派发至该车。
若希望车辆自动承接日常运输任务，请开启该开关。

6.5 车辆详情可用操作

在列表中点击「查看」进入详情。上方可查看告警、电量、负载、当前动作等信息；下方提供常用控制按钮：

按钮什么时候用
启用 / 禁用是否让该车参与作业
允许自动 / 禁止自动是否允许系统自动派单、自动充停
去充电在图上选一个充电点，立刻让车去充电
去停车在图上选一个停车点，立刻让车去停靠
发送 Instant Action立刻发一个动作（不走完整运输单）
发送 Order给这台车指定起点、终点，直接下发订单
重置锁点车辆占用点位无法释放、调度受阻时，尝试清理占用
配置路线给车设一条固定路线：起点 → 途经点 → 终点，并可设重复次数后发送
启用 / 停用 AGV 模拟用于联调演示。正式产线请勿随意开启。停用模拟时，系统还可能顺带重置锁点
详情里还可以看「当前 Action / 历史 Action」、负载信息；部分状态支持「简单 / 专家」两种查看方式。 编辑车辆参数通常要先停用车；车正在执行任务时一般不能停用；换地图前也要先停用。
如果车处于手动模式，系统会提示无法创建订单。请先在车端切回可自动执行的模式。

07订单：下发与跟踪

入口：左边菜单 订单管理 → 订单列表

订单是下发给车辆的工作指令：起点与终点、执行车辆，以及当前执行状态。

7.1 查询订单

操作步骤
  1. 先选择一张已启用的地图（必选，未选择则无法查询）。
  2. 需要时再按订单号、日期、车辆继续筛选。
  3. 点「查询」。
  4. 点某条订单的详情，可看路径、动作、执行过程；执行成功的订单还可以做「路径回溯」回放。

列表里还能看到订单来源，方便值班对账：用户创建、WMS、系统创建、前置订单等。

7.2 创建运输任务

操作步骤
  1. 在订单列表点击「新增 WMS 订单」。
  2. 选择地图、可用车型；如需指定车辆，再选择具体 AGV（也可不指定，由系统分配）。
  3. 选择优先级：
    • 立即执行：优先插入执行
    • 执行完当前订单后立即执行：当前订单完成后立即执行本单
    • 系统分配：按系统默认排队（进入页面时的常用选项）
  4. 在地图上选好起点、终点；需要时可添加途经点，并给起终点配置动作。
  5. 点「保存」。成功会提示「订单创建成功！」
创建前请确认：地图已启用、起点终点有效且不是同一个点、车型已选。 指定了车辆 → 订单一般进入「待执行」；没指定车辆 → 进入「待分配」。 使用「新增 WMS 订单」时，请确保接单车辆已开启「自动工作许可」，否则可能无法派发。

7.3 订单状态说明

状态含义说明
待分配订单已创建，等待系统分配车辆
待执行已分配车辆，等待开始执行
工作中车辆正在执行该订单
执行成功订单已完成
已取消 / 订单已关闭订单已终止，不再执行
执行失败 / 前置失败 等执行异常，请查看详情确认原因后，再决定重发或处理现场

7.4 取消、删除、复制

  • 取消：待分配、待执行、工作中的订单，一般可以取消。已经结束的订单通常不能再取消。
  • 删除：还在进行中的订单（待分配/待执行/工作中），列表里通常不能直接删；请先取消或等结束后再处理。
  • 复制：勾选订单后可按次数批量复制，适合重复性任务。

7.5 其他下单方式

  • 在某台车的详情里点「去充电 / 去停车 / 发送 Order」——适合针对单车快速处理。
  • 「Mock订单」用于测试批量造单，正式生产环境请勿使用。

常见订单类型：普通订单、充电订单、停车订单。 系统还可能自动生成「前置订单」「避让订单」，一般不用您手动建，列表里看到即可。

08看板：实时监控

入口：左边菜单 地图看板（通常新开全屏窗口）

看板为值班监控大屏，用于查看车辆位置、运行状态及是否存在告警。

操作步骤
  1. 进入后先选择一张已启用的地图。
  2. 中间看地图和车辆位置；右边看 AGV 列表卡片。
  3. 右上角可开关显示：车体安全区、电量、导航线、透明底盘等。
  4. 要换另一张图监控时，点设置图标重新选地图。

右侧卡片上，除了状态（工作中、空闲、去充电、充电中、报错、离线、未启用等）， 通常还能看到：正在执行的订单、动作数量、今日成功/失败数、当前位置。 多项信息可继续点击进入订单或车辆详情。

如果提示「地图未启用」或「当前无可用地图」，请先回到地图列表，把至少一张地图启用。
建议值班岗位保持看板常开，以便随时掌握现场情况。

09货架资料

入口：左边菜单 货架管理 → 货架列表

此处登记货架型号资料（名称、尺寸、最大载重等），供业务配置引用。 本页不是「现场各货架实时位置」的监控画面。

新增方法
  1. 点「创建货架」。
  2. 填写货架型号、名称，以及长/宽/高（毫米）、最大载重（公斤）等。
  3. 确认保存。之后需要修改可点编辑；不用了可删除（删除前请确认现场已不再使用该型号）。

10查日志

入口：左边菜单 日志管理 → 日志列表

出现异常需追溯系统记录时，可在此按时间查询。

操作步骤
  1. 选择日期范围（可用「近7天」等快捷选项）。
  2. 点「查询」。
  3. 需要时按类型（信息/警告/错误）、来源、模块继续筛选。
  4. 点「详情」查看完整内容。

11用户与权限（管理员）

入口：右上角头像 → 账号管理 / 角色管理 / 权限管理

  • 账号管理：给同事开通或停用登录账号。
  • 角色管理：决定某个岗位能看哪些菜单、能点哪些按钮。
  • 权限管理：查看或维护权限项。
建议仅向少数管理员开通用户管理；普通操作员按岗位最小化授权，以减少误操作。

12常见问题

无法打开地图看板？

请先确认至少有一张地图已启用；再检查浏览器是否拦截了新窗口。

无法编辑地图？

请先停用地图，再编辑并保存，最后重新启用。停用失败时，请确认是否仍有启用中的车辆，或尚未完成的订单。

车辆启用失败？

请按顺序检查：地图是否已启用、车辆是否在线、位置是否已上报并停在地图点上。

已下单但车辆未执行？

请依次确认：车辆是否已启用；是否处于手动模式；「自动工作许可」是否已开启（WMS 订单尤其关键）； 电量是否过低；是否暂停或报错；是否正在充电且电量尚未达到可接单水平；车辆是否仍载货（载货时系统通常不会自动派发下一单）。

去充电 / 去停车失败？

请确认地图中存在可用的充电点或停车点，并在弹窗中选定目标点后再发送。

提示没有权限？

联系管理员，在角色管理里给您的岗位开通对应菜单或按钮权限。

安全提醒

  • 改图、停用车辆前，先确认现场安全和相关任务已处理妥当。
  • 「重置锁点」会影响调度，确认异常后再点。
  • 「AGV 模拟」「Mock订单」只用于测试，不要在正式产线使用。

13服务器离线部署

现场需先在服务器完成 RCS 安装，方可打开管理端并使用本手册中的功能。 请先选择服务器操作系统，查看对应说明。当前一键安装包面向 Ubuntu（x86_64）Linux 服务器。

交付文件名类似 rcs-offline-linux-amd64-xxx.run（xxx 为版本号）。 推荐系统：Ubuntu 20.04 / 22.04 / 24.04，CPU 架构为 x86_64。

一键安装
  1. 将安装包上传至服务器，并进入该文件所在目录。
  2. 执行以下命令（请将文件名替换为现场实际交付的文件名）：
sudo bash rcs-offline-linux-amd64-xxx.run
Docker 环境： 安装脚本会自动检测 Docker Engine 与 Docker Compose 是否可用。 若未安装或不可用，脚本可提示并通过 Docker 官方软件源在线安装（需服务器可访问外网）。 纯离线机房请由管理员预先安装好 Docker。

脚本将自动解压、导入镜像并启动服务。完成后请按提示使用浏览器打开管理端地址登录。 更详细的端口、备份与升级说明，见交付包内的《离线部署》说明文档。

注意： 命令中的 xxx 请替换为实际版本号（例如交付文件上的日期编号），请勿照抄字母 xxx。

当前交付的一键安装包不支持在 Windows 本机直接部署 RCS 服务端。

  • 请使用一台 Ubuntu（x86_64）Linux 服务器完成部署，命令见「Linux」页签。
  • 部署完成后，可在 Windows 电脑上用浏览器访问管理端（默认 http://服务器IP）。
客户端操作系统与服务端操作系统相互独立：Windows 用于日常操作即可，服务请安装在 Linux 服务器上。

当前交付的一键安装包不支持在 macOS 本机直接部署 RCS 服务端。

  • 请使用一台 Ubuntu（x86_64）Linux 服务器完成部署，命令见「Linux」页签。
  • 部署完成后，可在 macOS 上用浏览器访问管理端（默认 http://服务器IP）。
现场生产环境请以 Linux 服务器为准；macOS 一般仅用于办公访问管理端。
